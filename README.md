# chinese-anti-ai-longform-rewrite

中文长文降 AI 味与去模板化改写 skill。

这个 skill 适合处理公众号长文、观点稿、整理稿等中文正文，重点删除对照句、教程感、机器式能力清单、抽象过渡句、空泛评价词和高频“把”字句，尽量只保留真正有信息量的观点句与信息句。

## 仓库内容

- `SKILL.md`：完整规则、禁用项、替换规则和操作顺序

## 适用场景

- 中文长文去 AI 味
- 公众号正文去模板感
- 观点稿、整理稿二次精修
- 需要去掉“讲解过程感”和“教程口吻”的正文改写

## 核心目标

- 先删模板，再保留内容
- 只留观点句和信息句
- 少解释，少过渡，少铺垫
- 让成稿更像成文，不像机器生成的讲解稿

## Before / After 示例

下面这个例子故意保留了几种常见的 AI 痕迹，包括对照句、教程口吻、抽象过渡句和高频“把”字句。

`Before`

```text
这套方法不是简单地把文章改短，而是把原本分散的内容整理成更清楚的结构。你可以先把重点提炼出来，然后再把重复的表达删掉。这样一来，整篇文章就可以变得更好读。它还可以帮助你把原本比较抽象的说法变成更容易理解的表达。
```

`After`

```text
这套方法先提炼重点，再删掉重复表达，让文章结构更清楚，也更好读。原本抽象的说法会被改成更直接、更容易理解的表达。
```

这个 skill 处理的重点通常是：

- 删掉“不是……而是……”这类对照模板
- 去掉“你可以”“这样一来”这类讲解口吻
- 压缩连续解释句，只保留真正有信息量的结果句

## 安装

如果你的环境支持 `skills add`，可以直接安装这个仓库：

```bash
npx skills add VHPSY/chinese-anti-ai-longform-rewrite -g -y
```

也可以手动把 `SKILL.md` 放到本地 skill 目录使用。

## 使用方式

1. 在需要处理中文长文时触发这个 skill。
2. 先按禁用项删除模板化表达。
3. 再按替换规则把句子改成更直接的陈述句。
4. 最后检查是否还残留抽象过渡、空泛评价和过密转场。

## 结果标准

- 读起来像成稿，不像说明书
- 句子里少见模板痕迹
- 每段只保留有信息量的句子
- 不靠过渡词撑结构
- 不靠抽象词堆气势
