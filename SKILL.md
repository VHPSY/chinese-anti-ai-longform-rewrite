# 中文长文反模板改写

## 适用场景

- 公众号长文
- 观点稿
- 整理稿
- 需要去 AI 味、去模板感、去教程感的中文正文

## 核心目标

- 先删模板，再保留内容
- 只留观点句和信息句
- 少解释，少过渡，少铺垫
- 让文章读起来像成稿，不像讲解过程

## 禁用项

### 1. 对照模板句

- `不是……而是……`
- `不只是……而是……`
- `不在于……而在于……`
- `这意味着……`
- `可以理解成……` 连用

### 2. 第二人称教程感

- `你可以……`
- `你还可以……`
- `这时你就可以……`
- `你想让……`
- `你让它……`
- `如果你……就……`

### 3. 机器式能力清单

- `它可以……`
- `它会……`
- `它还能……`
- `它甚至可以……`
- `再往后，它还可以……`
- 连续几句都以 `它` 开头

### 4. 抽象过渡句

- `比如……`
- `这样一来……`
- `另一个值得说的点……`
- `这类……`
- `这些……`
- `这种……`
- `后半段的演示，基本就是……`
- `最基础的用法……`
- `最值得注意的地方……`
- `做法并不复杂……`

### 5. 空泛评价词

- `很大`
- `很明显`
- `很实用`
- `很关键`
- `很有意思`
- `真正`
- `实际`

### 6. 过密转场

- `接着`
- `然后`
- `之后`
- `再往后`
- `最后`
- `同时`
- `于是`
- `所以`

### 7. 抽象名词堆叠

- `流程`
- `工作流`
- `能力`
- `工具`
- `入口`
- `集成`
- `方案`
- `价值`
- `优势`

### 8. 解释性冗句

- `它的意思是……`
- `这说明了一件事……`
- `这就带出了……`
- `这个功能看起来小，实际很好用`
- `如果做到后面，你会发现……`

### 9. 高风险“把”字句

优先删除：

- `把……变成……`
- `把……整理成……`
- `把……做成……`
- `把……放进……`
- `把……接上……`
- `把……串起来……`
- `把……带进……`
- `把……发出去……`
- `把……生成……`

谨慎保留：

- `把邮件发出去`
- `把文件收在一起`
- `把任务交出去`
- `把数据放进数据库里`

## 优先替换规则

### 1. 第二人称改陈述

- `你可以` → 删除“你”
- `你还可以` → 删除“你还”
- `这时你就可以` → `这时可以`
- `你想让` → `想让`
- `你让它` → 直接说动作

### 2. 对照句改直说

- `不是 A，而是 B` → 能删 A 就删 A，只留 B
- `不只是 A，而是 B` → 常常直接写 B
- `不在于 A，而在于 B` → 直接写 B
- `这意味着` → 直接写结果
- `可以理解成` → 直接写定义

### 3. 机器式能力句改并句

- `它可以……它可以……它还能……` → 合并成一两句
- `它会……它会……` → 留最关键的一个
- `它甚至可以……` → 多半删掉“甚至”
- `再往后，它还可以……` → 多半删掉整句

### 4. 抽象过渡句改结果句

- `比如……` → 直接举例，不先打招呼
- `这样一来……` → 删掉，保留结果
- `另一个值得说的点……` → 直接进入事实
- `这类……`、`这些……`、`这种……` → 能具体就具体

### 5. “把”字句改写

- 能前置动词就前置
- 能直接说结果就直接说结果
- 一句里尽量别连续出现两个以上“把”字结构
- 标题、导语、结尾优先少用“把”字句

## 操作顺序

1. 先删对照模板
2. 再删第二人称引导
3. 再压能力清单
4. 再删过渡句
5. 再扫泛指词
6. 再收紧“把”字句
7. 最后只留观点句和信息句

## 结果标准

- 读起来像成稿，不像说明书
- 句子里少见模板痕迹
- 每段只保留有信息量的句子
- 不靠过渡词撑结构
- 不靠抽象词堆气势
- 不靠“把”字句制造流程感
